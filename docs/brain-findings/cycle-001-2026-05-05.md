## Discovery: Twin Architecture Requires Credential Bootstrapping

The twin-loop architecture assumes NAS is always mounted. In practice, macOS SMB mounts can go stale (observed: "Interrupted system call" on a previously-working mount). The infrastructure needs:
- A pre-cycle mount check with keychain-based auto-remount
- A local fallback directory (~/ARES_Brain/) for operation when NAS is down
- Queued writes that sync to NAS when mount returns

This is an infrastructure hardening task for a future cycle.
