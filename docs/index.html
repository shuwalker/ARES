Architecture

What ARES actually is

ARES does not replace JaegerAI or Hermes. It composes them behind one interface. The runtime owns the turn. ARES owns the presentation.

⚔️ The Name

Artificial Reasoning & Execution System

ARES is the client and integration layer. It is not a second brain, a second memory store, or a company simulator. It is the UI, adapter host, and native app shell that makes independent runtimes feel like one assistant.

💻 Mac First

Native App + Web Everywhere

The SwiftUI Mac app wraps the Web UI in a WKWebView with native menus, system integration, and server control. The same Web UI is reachable from phones, tablets, and other devices over Tailscale or LAN.

🔄 Runtime Switching

Session-Scoped Backends

Each chat session remembers whether it uses JaegerAI, Hermes, or Hybrid. The global default sets the starting point; the session selector overrides it per conversation. No restart required.

🤖 Embodiment Path

JaegerAI Is the Required Runtime

JaegerAI handles the agent loop, memory, characters, voice, tools, skills, and hardware abstraction. ARES detects an existing JaegerAI install or delegates to JaegerAI's own installer. Without JaegerAI, there is no Companion.

Capabilities

What works now

Verified, tested, and committed. Not roadmap fiction.

🧠

Identity Projection

ARES builds a runtime-aware identity prompt for every turn. The model knows which backend is active, what tools are available, and what the Companion's name is.

🌐

Web UI

Self-contained Python server with streaming SSE, session management, password auth, and hot reload. Edit Python files and the server restarts in ~2s.

🔄

Backend Selector3 modes

JaegerAI (required Companion), Hermes (optional coding/terminal addition), or Hybrid (compose capabilities deliberately). Set per session.

🎭

Character Avatar Browser

14 visual personas with card art, traits, and lore. HAL 9000, GLaDOS, Jarvis, TARS, Bender, Helldiver, and more. Reads from JaegerAI character data.

💻

Native macOS App

SwiftUI shell with WKWebView, native menus, server control, and device mesh status. Swift build is clean. Tabbed interface for Chat, Hermes, and JaegerAI terminals.

🏢

Device Meshnew

Primary AI body (Mac Studio) vs joined devices (MacBook Pro). Each device advertises capabilities. Offline mode with sync-back journal.

🗣️

Voice I/O

Edge-TTS for speech output. Whisper STT for speech input. Optional Kokoro TTS via Hermes tools. Runs locally or delegates to JaegerAI voice pipeline.

⚡

Hot Reload

Edit static files and the browser reloads automatically. Edit Python and the server restarts. Zero manual intervention for frontend or backend changes.

🔒

Private by Default

Runs on your machine with local models via JaegerAI or cloud providers you configure. No telemetry. No external dependency unless you opt in.

Install

Run it locally

One command. JaegerAI installs automatically. Hermes is optional.

bash 
# Clone and install
git clone https://github.com/shuwalker/ARES.git
cd ARES
bash install.sh

# Optional: also install Hermes Agent for coding/terminal work
bash install.sh --with-hermes

# Run
./start.sh
# → http://localhost:8787

# Or launch the native Mac app
swift run ARES
