# ARES Repo Evaluation

**Criteria:** Does this repo help build an all-in-one AI tool that learns to use tools and supplements human tasks to improve human output?

## Tier 1: Directly Portable — Fits ARES's Needs

### Paperclip (1701 ts)
**What it is:** Multi-agent orchestration platform for organizations. Agents as employees with managers, org charts, teams.
**What ARES needs:** Adapter registry, task tracking, approval queue, routines, skill studio, agent config, multi-agent chat.
**Verdict:** PORT. Strip company scoping. Keep: adapter registry pattern, Issues/kanban, Inbox/approvals, Routines, SkillStudio, Secrets, AgentDetail, BoardChat. This is the closest codebase to what ARES needs — just reframe from "company" to "synthetic person."

### Hermes WebUI (1104 ts)
**What it is:** Web dashboard for managing one Hermes Agent instance. Chat, sessions, files, analytics, models, cron, skills, plugins, MCP, channels, credentials, system health, i18n.
**What ARES needs:** Model picker, cron, skills editor, credentials, channels, plugin system, system health, i18n.
**Verdict:** PORT. Re-architect from per-instance to per-agent. The model picker, cron, skills, credentials, channels, and plugin system are all directly portable. The i18n system (15 languages) is a bonus.

### Hermes Workspace (815 ts)
**What it is:** React dashboard/IDE for Hermes with gateway, workspace, sessions, and a swarm.yaml worker pattern.
**What ARES needs:** Workspace UI patterns, gateway architecture, worker orchestration.
**Verdict:** PORT selectively. The workspace UI patterns and swarm.yaml worker model are directly applicable. The gateway architecture is reference for how ARES should handle multi-agent routing.

### SAM — Synthetic Autonomic Mind (201 ts + 440 py + 272 swift)
**What it is:** Native macOS AI assistant with multi-provider API framework, AgentOrchestrator, MCP framework, voice pipeline, conversation engine with memory.
**What ARES needs:** Voice pipeline, multi-provider API framework, AgentOrchestrator, MCP framework, conversation memory.
**Verdict:** PORT patterns. The multi-provider API framework (OpenAI, Anthropic, Gemini, DeepSeek, MiniMax, OpenRouter) is directly applicable. The AgentOrchestrator for multi-step workflows is the blueprint for ARES's routing. The voice pipeline is Phase 4. The MCP framework with 8 tools/60+ operations is reference.

### gbrain (1969 ts)
**What it is:** Knowledge graph brain layer for AI agents. Entity extraction, synthesis, graph traversal, gap analysis. 146K+ pages indexed.
**What ARES needs:** Persistent memory/context store that tracks temporal relationships.
**Verdict:** PORT patterns. The temporal knowledge graph architecture is exactly what ARES's Context Store should be. Entity extraction from conversations, synthesis patterns, and gap analysis are the blueprint. This replaces ARES's current stub context_store.py.

### Agent Governance Toolkit (342 ts + 1977 py)
**What it is:** Runtime governance for AI agents: policy enforcement, zero-trust identity, execution sandboxing, SRE, compliance.
**What ARES needs:** Approval/policy engine for gating agent actions.
**Verdict:** PORT patterns. The approval/policy engine and human-in-the-loop patterns are directly applicable to ARES's intervention points. The `await_human` pattern is how ARES should pause for approval.

### Open WebUI (75 ts + 237 py)
**What it is:** Most popular open-source LLM UI. Multi-model chat, RAG, document upload, model management, voice, image gen.
**What ARES needs:** Model management UI, document/RAG upload, multi-model conversation patterns.
**Verdict:** PORT patterns. The model management UI (switching between providers mid-conversation) and document/RAG upload patterns are directly applicable. Svelte-based so no direct code porting.

### Open-LLM-VTuber (125 py)
**What it is:** Voice-interactive AI companion with Live2D avatar support. Modular engine system (LLM, ASR, TTS, VAD). WebSocket-based.
**What ARES needs:** Voice pipeline, avatar/face system, real-time audio streaming.
**Verdict:** PORT patterns (Phase 4). The voice pipeline architecture (ASR → LLM → TTS with real-time streaming and interruption) and character/avatar config system are the blueprint for ARES's voice and face.

### colibri (just pulled, C)
**What it is:** Pure C MoE inference engine for GLM-5.2 744B model on consumer hardware (~25GB RAM). Expert disk streaming.
**What ARES needs:** Local inference capability for running large models on the Mac Studio.
**Verdict:** REFERENCE. Not directly portable (C code), but the expert disk streaming pattern is relevant for running large models on 32GB RAM. The OpenAI-compatible server interface is useful.

## Tier 2: Reference Only — Relevant Patterns, Not Direct Porting

### Hermes Desktop (153 swift)
**What it is:** Native macOS companion for Hermes Agent.
**What ARES needs:** Native macOS patterns, subagent tree viewer, session management.
**Verdict:** REFERENCE. Swift code, not directly portable. The subagent tree and session management patterns are worth studying.

### hermes-paperclip-adapter (12 ts)
**What it is:** Adapter that makes Hermes work as an employee in Paperclip.
**What ARES needs:** Adapter interface contract.
**Verdict:** REFERENCE. Shows the Paperclip side of the adapter contract. Already used as reference for ARES's adapter pattern.

### Lilith-AI (108 py)
**What it is:** Local-first personal cognition system for Jenkins Robotics. Desktop assistant on macOS, later Jaeger robot fleet.
**What ARES needs:** Personal cognition system patterns.
**Verdict:** REFERENCE. Same domain as ARES (personal AI). Worth reading for architecture patterns, but it's a separate project with different goals (robotics embodiment).

### aiavatarkit (211 py)
**What it is:** AI-based conversational avatar building SDK.
**What ARES needs:** Avatar/face system.
**Verdict:** REFERENCE (Phase 4). Avatar rendering SDK. Worth looking at when ARES gets a face.

### hermes-example-plugins (3 py)
**What it is:** Reference plugin implementations for Hermes Agent.
**What ARES needs:** Plugin system reference.
**Verdict:** REFERENCE. Shows the Hermes plugin API. Useful when building ARES's plugin system.
